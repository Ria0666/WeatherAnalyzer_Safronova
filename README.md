Weather Analyzer

Информационно-аналитическая система для загрузки, хранения, обработки, анализа и визуализации исторических метеорологических данных.

Проект реализован на C++ с использованием PostgreSQL. В качестве открытого источника данных используется Open-Meteo Historical Weather API.

1. Описание проекта

Weather Analyzer — консольное приложение, которое позволяет загружать исторические погодные данные по выбранным городам, сохранять их в базу данных, выполнять аналитическую обработку, находить температурные аномалии, строить простые прогнозы и формировать итоговый PDF-отчёт.

Система предназначена для анализа следующих погодных параметров:

* температура воздуха
* влажность
* атмосферное давление
* скорость и направление ветра
* количество осадков
* облачность

В проекте реализованы основные этапы работы с данными: получение из открытого API, проверка корректности, сохранение в PostgreSQL, аналитическая обработка, прогнозирование и представление результатов в виде отчёта.

2. Основные возможности

Система предоставляет следующие функции:

* загрузка исторических погодных данных из Open-Meteo
* сохранение данных в PostgreSQL
* проверка корректности загружаемых значений
* логирование процесса импорта
* расчёт средней, минимальной и максимальной температуры
* анализ температуры по месяцам
* анализ количества осадков
* подсчёт дней с осадками
* сравнение городов по средней температуре
* поиск температурных аномалий
* прогноз средней температуры
* прогноз вероятности осадков
* формирование итогового PDF-отчёта с таблицами и графиками

PDF-отчёт содержит:

* графики изменения температуры
* графики осадков
* сравнительные диаграммы по городам
* таблицы аналитических показателей
* список найденных аномалий
* результаты прогноза (строится на 7 дней после конца выбранного периода)
* итоговый аналитический вывод

4. Архитектура приложения

Проект разделён на несколько логических модулей.

Database. Класс Database отвечает за подключение к PostgreSQL через библиотеку libpqxx.

OpenMeteoClient. Класс OpenMeteoClient выполняет HTTP-запросы к Open-Meteo API, получает JSON-ответ и преобразует его в объекты погодных наблюдений.

Repository-классы. Repository-классы отвечают за работу с отдельными таблицами базы данных:
* CityRepository — получение городов
* StationRepository — создание и получение метеостанций
* ObservationRepository — сохранение погодных наблюдений
* ImportLogRepository — запись логов импорта

ImportService. ImportService управляет процессом загрузки данных:
1. получает список городов
2. загружает данные из Open-Meteo
3. проверяет корректность значений
4. сохраняет наблюдения в базу данных
5. записывает результат импорта в журнал

AnalyticsService. AnalyticsService выполняет аналитические SQL-запросы:
* расчёт средней, минимальной и максимальной температуры
* расчёт средней температуры по месяцам
* анализ осадков
* подсчёт дней с осадками
* сравнение городов

AnomalyService. AnomalyService используется для поиска температурных аномалий.

Аномалии определяются на основе дневных средних температур. Для выбранного города и периода строится линейная регрессия, после чего фактическая средняя температура дня сравнивается с ожидаемым значением. Если отклонение превышает заданный порог, день считается аномальным.

ForecastService. ForecastService реализует простые прогнозные модели на основе метода скользящего среднего. В системе реализованы два вида прогноза:
* прогноз средней температуры;
* прогноз вероятности осадков.

Для прогноза температуры система берёт среднюю температуру за несколько предыдущих дней и использует её как прогнозное значение. Для прогноза вероятности осадков система анализирует, в какой доле предыдущих дней были осадки, и переводит это значение в проценты.

ReportService. ReportService формирует итоговый отчёт. Сначала создаётся HTML-представление отчёта с таблицами и SVG-графиками, затем оно автоматически конвертируется в PDF с помощью Google Chrome Headless.
